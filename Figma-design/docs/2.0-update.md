# AI UI Asset Generator 2.0 更新说明

AI UI Asset Generator 2.0 是一次从“AI 生图导入 Figma 原型”到“AI 设计稿生成 + 切图资产管理 + Figma 工作流增强”的版本升级。

本版本继续保留 1.0 已跑通的核心链路：生成设计稿、手动切图、透明 PNG、保留坐标、放入 Figma。同时新增了更完整的供应商配置、切图编辑、资产处理和可编辑设计稿实验能力。

## 版本信息

- 版本号：2.0.0
- 运行环境：Figma Desktop 插件 + 本地 Node.js API 代理
- Node.js：20+
- 本地 API：`http://127.0.0.1:18787`
- 本地预览：`http://127.0.0.1:4173/figma-sim.html`

## 主要更新

### 1. 多供应商配置

2.0 新增供应商切换和本地配置保存能力：

- 第三方接口
- 官方 OpenAI API
- OpenRouter

OpenRouter 模型固定为：

```text
gpt-5.4-image-2
```

配置保存后不会把 API Key 写入源码，也不会提交到 GitHub。真实配置存储在本地 `.local-provider-config.json`，该文件已被 `.gitignore` 忽略。

### 2. 生成设计稿体验优化

- 按钮文案调整为 `生成设计稿`
- 支持文生图和图生图
- 图生图支持上传、拖拽、粘贴多张参考图
- 支持多图生成结果
- 支持轮播切换当前设计稿
- 支持单张和全部下载
- 生成图展示区支持更稳定的等比预览和滚动查看

### 3. 插件窗口体验升级

- 支持插件窗口拖拽调整大小
- 内容区域自适应不同屏幕尺寸
- 支持最小化窗口
- 右侧内容超出时支持整体滚动
- 切图设置面板支持拖动，避免遮挡目标区域

### 4. 切图能力增强

2.0 将切图从“一次性框选”升级为“可编辑资产区域”：

- 在生成图上拖拽创建切图区域
- 切图区域支持移动
- 切图区域支持缩放
- 点击切图名称可打开设置面板
- 支持精确设置：
  - X 坐标
  - Y 坐标
  - 宽度
  - 高度
  - 圆角
- 支持删除切图
- 切图资产保留原图坐标和尺寸
- 每张生成图的切图资产相互隔离，避免多图切换时混用

### 5. 切图资产处理

每个切图资产支持多种处理方式：

- 原始 PNG
- 透明 PNG
- AI 透明 PNG
- SVG 转换
- AI 重绘 SVG

其中 SVG 仍属于实验能力。当前更推荐把 SVG 作为“可尝试的增强选项”，PNG 仍是稳定资产格式。

### 6. 切图后背景处理

2.0 增加了切图后原图区域的弱化 / 修复能力：

- 切图资产被提取后，原图区域可以做柔和填充
- 避免原图上已切出的元素继续强烈显示
- 保留切图示意框，不影响用户继续调整区域

这部分属于视觉修复能力，后续还可以继续提升边缘自然度。

### 7. 放入 Figma 流程调整

`放入 Figma` 现在收敛为两条路径：

1. `源文件`
2. `编辑设计稿`

#### 源文件

稳定路径。保持当前逻辑不变：

- 创建 Figma 画布
- 放入生成设计稿原图
- 放入切图资产
- 保留切图资产坐标和尺寸
- 保留 PNG / 透明 PNG / SVG 等资产结果

#### 编辑设计稿

实验路径。用于探索“AI 图片 -> HTML 预览 -> Figma 可编辑图层”的工作流：

- 根据当前生成图和切图资产生成 HTML 预览
- 支持先查看还原效果
- 再将预览内容导入 Figma
- 导入时尝试保留文字、图片、形状、圆角、阴影、基础渐变等信息

旧版 `可编辑设计稿（实验）` 入口已隐藏，避免和新的 `编辑设计稿` 路线混淆。

### 8. HTML 到 Figma 实验链路

2.0 引入了 web-to-figma 思路：

- 在 H5 预览 iframe 中捕获 DOM
- 获取元素位置、尺寸、文本、图片、样式和 computed style
- 映射成插件可创建的 Figma 节点
- 尝试还原成可编辑 Figma 图层

项目中新增：

```text
vendor/web-to-figma-capture-adapter.js
```

该文件用于承接 HTML DOM 捕获和转换实验。

### 9. 字体优化

导入 Figma 时对文字字体做了优化：

- 中文优先使用 `PingFang SC`
- 数字 / 金额 / 数据类文本优先使用 DIN 系列字体
- 如果本机或 Figma 环境没有对应字体，会回退到可用字体

## 使用方式

### 安装依赖

```bash
npm install
```

### 启动本地 API

```bash
npm run api
```

### 启动本地预览

```bash
npm run preview
```

打开：

```text
http://127.0.0.1:4173/figma-sim.html
```

### 在 Figma 中运行

1. 打开 Figma Desktop
2. 进入 `Plugins > Development > Import plugin from manifest...`
3. 选择项目里的 `manifest.json`
4. 运行 `AI UI Asset Generator`
5. 在插件配置中填写供应商、Base URL、模型和 API Key
6. 生成设计稿
7. 进入切图模式
8. 选择 `源文件` 或 `编辑设计稿` 放入 Figma

## 安全注意事项

提交 GitHub 前不要提交以下内容：

- `.env`
- `.env.local`
- `.local-provider-config.json`
- 任何真实 API Key
- `node_modules/`
- 临时日志和生成文件

当前 `.gitignore` 已包含这些敏感配置文件。
