# 产品愿景

Image-to-Figma Design 要解决的问题很窄：把一张清晰 PNG 截图或设计稿，快速转换成 Figma 画布里的可编辑设计稿。

## 一句话定义

用户上传单张 PNG 后，系统生成 DSL v0.1，Figma 插件用 Renderer 把 DSL 写成接近原图、主要内容可编辑的 Figma 图层。

## 目标用户

- 设计师：把截图或旧稿快速转成可继续编辑的 Figma 草稿。
- 设计工作室：降低重复复刻页面的时间成本。
- 前端开发者：把参考截图转换成可检查结构的设计稿。
- 独立开发者：快速把 App、小程序或简单后台截图变成可迭代稿。

## 一期成功判断

一期不追求商业化完整产品。它只证明这条链路值得继续投入：

```text
PNG 上传
-> 后端识别
-> DSL v0.1
-> Figma Renderer
-> Figma 可编辑设计稿
```

成功标准是：

- 大部分样例能生成，不因为局部复杂区域整页失败。
- 主要文字可编辑。
- 主要图片能回填到接近原图的位置。
- 基础形状、边框、圆角、分割线能表达页面结构。
- 复杂 Banner、插图、图表、多色图标可以 fallback 为图片。
- 失败时能定位到任务阶段和错误码。

## 最高原则

先生成得出来，再追求生成得更好。

一期优先级：

1. 稳定生成。
2. 视觉接近。
3. 主要内容可编辑。
4. 速度可接受。
