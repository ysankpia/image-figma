# UI 指南

插件 UI 是工具，不是营销页。

## Principles

- 简洁、直接、少状态。
- 不暴露内部术语。
- 不做大面积装饰。
- 优先清楚地告诉用户当前发生什么。
- 错误文案面向用户，调试信息面向开发者。

## Views

只保留：

- UploadView。
- PreviewView。
- ProgressView。
- DoneView。
- ErrorView。

不要加：

- 首页营销介绍。
- 历史任务。
- 设置页。
- 账号页。
- 质量报告。
- 多模型选择。

## Text

用户可见文案使用：

- 上传中。
- 处理中。
- 生成中。
- 正在写入 Figma。
- 生成完成。
- 生成失败。

避免：

- OCR。
- DSL。
- AI 分析。
- 模型调用。
- 置信度。
- 质量评分。

## Layout

- 插件窗口内容要紧凑。
- 按钮要明确主次。
- 错误态要给用户下一步。
- 文件信息只显示名称、尺寸、大小和风险。
- 不在 UI 中解释技术架构。
