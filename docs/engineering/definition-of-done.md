# Definition of Done

一个任务完成必须同时满足实现、验证、文档三件事。

## Documentation Work

文档任务完成条件：

- 文档放在正确目录。
- `docs/index.md` 可导航到它。
- 相对链接可用。
- 没有模板占位符残留。
- 不和现有正式文档冲突。
- 如果替代旧草稿，旧草稿在 legacy 中可追踪。

## Implementation Work

后续实现任务完成条件：

- 代码实现符合当前计划。
- 相关 DSL、API、数据模型或用户流程文档已更新。
- 测试或验证证据已记录。
- 没有把 P1/P2 功能塞进 P0。
- 没有打破模块边界。

## Bug Work

Bug 修复完成条件：

- 有 bug 记录。
- 有复现说明。
- 有根因。
- 有修复说明。
- 有回归保护。
- 有验证证据。

## MVP Work

MVP 阶段完成条件：

- 主链路可跑通。
- P0 验收通过。
- 样例验收记录完整。
- 失败能定位阶段和错误码。
- 下一阶段遗留问题记录清楚。
