~~~markdown
# 一期 MVP 范围 v0.1

文档名称：一期 MVP 范围  
所属项目：Image-to-Figma Design  
当前版本：v0.1  
版本性质：MVP 开发准备版  
适用阶段：第一版核心链路开发  
最后更新：2026-05-16  

---

## 1. 文档目的

本文档用于明确 Image-to-Figma Design 项目第一版 MVP 的开发范围。

第一版不是完整商业化产品，也不是最终形态。第一版只验证一件事：

> 用户上传 PNG 截图后，系统能否快速生成接近原图的 Figma 可编辑设计稿。

所有和这条核心链路无关的功能，默认不进入 v0.1。

---

## 2. 一期 MVP 核心目标

一期 MVP 只追求四个目标：

1. 快  
2. 像  
3. 可编辑  
4. 稳定生成  

具体来说：

```text
PNG 上传
→ 后端识别
→ 生成 DSL v0.1
→ Figma 插件渲染
→ Figma 可编辑设计稿
~~~

第一版是否成功，不看功能有多全，而看这条链路是否能跑通、是否足够稳定、生成效果是否接近原图。

------

## 3. 一期 MVP 核心链路

一期核心链路如下：

```text
用户上传 PNG
↓
后端接收图片
↓
图片预处理
↓
OCR 文字识别
↓
AI / CV 结构分析
↓
图片资产裁切
↓
生成 DSL v0.1
↓
DSL 校验与基础修复
↓
Figma 插件获取 DSL
↓
Image-to-Figma Renderer 渲染图层
↓
生成 Figma 可编辑设计稿
```

其中最关键的两个技术核心是：

```text
1. DSL v0.1
2. Image-to-Figma Renderer
```

DSL 是后端和 Figma 插件之间的数据协议。
Renderer 是把 DSL 转成 Figma 图层的核心渲染包。

------

## 4. 一期 MVP 面向用户

一期 MVP 的主要用户是：

```text
设计师
设计工作室
前端开发者
独立开发者
```

用户典型需求：

```text
把 PNG 设计稿 / App 截图 / 小程序截图快速转成 Figma 可编辑稿
```

用户不需要理解 OCR、AI、DSL、模型调用等内部实现。

------

## 5. 一期 MVP 支持的输入

### 5.1 支持格式

一期只支持：

```text
PNG
```

暂不支持：

```text
JPG
JPEG
WebP
PDF
Figma 文件
Sketch 文件
PSD 文件
网页 URL
```

后续版本可扩展 JPG / WebP。

------

### 5.2 推荐输入类型

一期重点支持：

```text
App 移动端截图
小程序页面截图
移动端高保真设计稿 PNG
```

一期基础兼容：

```text
后台系统截图
简单 Web 页面截图
```

一期不重点优化：

```text
Landing Page 长网页
复杂后台报表
复杂图表页面
低清晰度截图
严重压缩图片
```

------

### 5.3 图片质量要求

一期不强制拒绝低质量图片，但需要提示用户。

推荐上传：

```text
清晰 PNG
高保真设计稿
移动端页面截图
文字清晰
无严重压缩
```

如果检测到以下情况，可提示风险但允许继续：

```text
图片尺寸过小
图片模糊
文字太小
压缩痕迹明显
图片过大
超长截图
```

------

## 6. 一期 MVP 必须支持的输出

一期最终输出是：

```text
Figma 当前画布中的可编辑设计稿
```

需要生成：

```text
Frame
Text
Shape
Image
Icon
Line
Group / Frame 层级
```

生成结果应具备：

```text
主要文字可编辑
主要图片回到对应位置
基础图标尽量矢量化
常见背景、圆角、阴影、边框可还原
复杂区域允许图片 fallback
原 PNG 作为隐藏参考层保留
```

## 7. 一期 MVP 必须包含的能力

### 7.1 Figma 插件最小能力

一期插件端只做最小功能：

```text
上传 PNG
图片预览确认
开始生成
显示简单进度
生成到 Figma 画布
生成完成提示
失败提示
```

不做复杂插件 UI。

进度展示只显示非技术化文案，例如：

```text
上传中
处理中
生成中
正在写入 Figma
生成完成
```

不向普通用户暴露：

```text
OCR
DSL
AI 分析
模型调用
Schema 校验
Fallback 细节
```

------

### 7.2 后端最小能力

后端一期必须支持：

```text
接收 PNG 上传
创建任务 taskId
保存原图
图片预处理
OCR 识别
AI / CV 分析
生成 DSL
保存 DSL
保存图片资产
提供 DSL 获取接口
提供图片资产访问接口
记录错误日志
```

开发阶段图片资源可以先走本地文件 URL。
生产阶段再切换 OSS / 对象存储 + 签名 URL。

------

### 7.3 DSL v0.1 能力

DSL v0.1 必须能表达：

```text
页面尺寸
页面背景
安全区
图层树
元素类型
元素坐标
元素样式
文本内容
图片资源
图标资源
隐藏原图参考层
Fallback 区域
基础 meta 信息
```

一期 DSL 元素类型只保留：

```text
frame
group
text
shape
image
icon
line
```

Button、Card、SearchBar、TabBar 等不作为独立 type，统一作为 role：

```text
type = frame
role = button / card / search_bar / tab_bar
```

------

### 7.4 Renderer 能力

Image-to-Figma Renderer 一期只负责：

```text
读取 DSL
校验基础字段
创建 Figma Frame
递归渲染 children
创建 Text
创建 Shape
创建 Image
创建 Icon
创建 Line
应用基础样式
插入隐藏原图参考层
处理 fallback 图片区域
```

Renderer 不负责：

```text
OCR
AI 分析
图片裁切
业务语义判断
质量评分
代码生成
Auto Layout 推断
组件化
```

Renderer 的核心原则：

```text
简单
稳定
快速
可维护
```

### 8.6 阴影

一期只支持常见单层阴影：



### 8.8 复杂区域 Fallback

一期必须支持 fallback。

以下情况优先 fallback：

```text
复杂 Banner
复杂插图
复杂图表
复杂运营图
复杂多色图标
复杂渐变光效
无法稳定识别的局部区域
```

原则：

```text
能编辑的编辑，复杂的先图片兜底。
```

## 11. 一期错误处理范围

一期必须做到：


## 15. 一期性能目标

一期要避免过度复杂导致生成太慢。

建议内部目标：

```text
简单移动端页面：15～30 秒
中等复杂页面：30～60 秒
复杂页面：60～90 秒
超时或失败：给出失败提示
```

速度优化原则：

```text
普通页面尽量 1 次主 AI 调用
异常页面最多 1 次 repair
复杂区域直接 fallback
不要生成过多节点
不要裁切过多图片
不要第一版做完整质量评分闭环
```

------

## 16. 一期必须避免的错误方向

第一版不能为了“完美可编辑”导致系统变慢、变复杂、不可控。

必须避免：

```text
每个细节都强行矢量化
每个组件都强行组件化
复杂 Banner 强行拆层
复杂图表强行还原
多轮 AI 反复修复
一开始就做 Auto Layout
一开始就做代码生成
一开始就做用户侧评测报告
```

第一版的正确策略是：

```text
主要内容可编辑
复杂区域图片兜底
视觉整体接近
生成速度可接受
链路稳定
```

------

## 17. 一期明确不做事项

以下内容不进入 v0.1：

```text

真正 Figma Component / Instance
一键组件化
Auto Layout 自动推断
插件内质量分
插件内详细评测报告
插件内原图 / 生成图对比
历史记录
最近任务
完整多模型对比平台
差异热力图
点击错误区域定位 elementId
完整评分看板
Web / Landing Page 深度支持
复杂图表结构化
复杂后台表格完美识别
用户账号系统
额度系统
支付系统
复杂缓存体系
自动切长图为多个页面
高级图标库
Assets 区域
```

这些功能可进入后续版本。

------

## 18. 一期成功标准

一期 MVP 可以认为成功，当满足以下条件：

```text
1. 用户可以上传 PNG
2. 后端可以生成 DSL v0.1
3. DSL 可以通过基础校验
4. Figma 插件可以成功渲染 DSL
5. Figma 里生成完整页面 Frame
6. 主要文字可编辑
7. 主要图片位置正确
8. 常见图标能部分矢量化
9. 复杂区域不会导致整页失败
10. 原图参考层默认隐藏保留
11. 生成速度在可接受范围内
12. 失败时有明确错误日志
```

## 20. 版本结论

v0.1 的核心不是做一个完整产品，而是做出一条可验证的主链路：

```text
PNG → DSL → Figma 可编辑设计稿
```

第一版必须克制。

优先保证：

```text
快
像
可编辑
稳定
```

暂时放弃：

```text
全功能
全自动
全可编辑
全组件化
全商业化
```

只有核心链路跑通后，后续功能才有继续开发的价值。
