# AI UI Asset Generator 使用说明与功能说明

版本：1.0

AI UI Asset Generator 是一个运行在 Figma 里的 AI UI 设计稿生成与切图工具。它的目标不是只生成一张图片，而是把“生成设计稿 -> 选择方案 -> 手动切图 -> 透明资产 -> 放入 Figma”这条流程尽量做顺。

## 一、工具能做什么

### 1. 生成 UI 设计稿

你可以输入一句提示词，让模型生成 App 首页、弹窗、活动页、工具页、会员页、游戏页、电商页等 UI 视觉稿。

支持两种生成方式：

- 文生图：只输入描述词生成设计稿
- 图生图：上传、拖拽或粘贴参考图，再基于参考图生成新的设计稿

### 2. 多张设计稿选择

生成多张图片时，插件会保留多个结果。用户可以通过轮播切换不同设计稿，选择满意的一张再进入切图流程。

### 3. 手动切图

进入切图模式后，可以直接在生成图上框选元素，例如：

- icon
- 按钮
- 插画
- 卡片
- banner
- 装饰元素
- 局部 UI 模块

每个切图都会记录它在原始设计稿中的坐标、宽高和顺序。

### 4. 透明底资产

切出的元素可以处理为透明底 PNG，方便后续在 Figma 中单独移动、组合和复用。

### 5. 放入 Figma

点击 `放入 Figma` 后，插件会在 Figma 中创建画布，并把生成的设计稿和切图资产放入画布中。

当前 1.0 版本重点是跑通图片生成、切图、透明 PNG 和 Figma 导入流程。

## 二、适合谁用

这个工具适合：

- UI 设计师：快速探索不同 App 界面方向
- 产品经理：快速生成原型视觉参考
- 独立开发者：生成 App 设计稿和可复用素材
- 运营设计：快速拆分活动页、弹窗、图标等素材
- AI 工作流玩家：把 AI 生图接入 Figma 设计流程

## 三、本地运行方式

### 1. 安装依赖

在项目目录执行：

```bash
npm install
```

### 2. 配置环境变量

复制示例文件：

```bash
cp .env.example .env
```

然后在 `.env` 或插件设置面板里填写 API 配置。

注意：真实 API Key 不要提交到 GitHub。项目已经默认忽略 `.env` 和 `.local-provider-config.json`。

### 3. 启动本地 API 服务

```bash
npm run api
```

默认地址：

```text
http://127.0.0.1:18787
```

如果插件里提示 `Failed to fetch`，通常就是这个服务没有启动，或者 Figma 插件访问不到本地服务。

### 4. 启动本地预览

```bash
npm run preview
```

浏览器打开：

```text
http://127.0.0.1:4173/figma-sim.html
```

这个页面用于模拟 Figma 插件运行环境，方便开发和调试。

## 四、在 Figma 中使用

### 1. 导入插件

1. 打开 Figma Desktop
2. 进入 `Plugins > Development > Import plugin from manifest...`
3. 选择项目根目录下的 `manifest.json`
4. 运行 `AI UI Asset Generator`

### 2. 配置模型服务

点击插件里的设置按钮，选择供应商并填写配置。

当前支持：

- 第三方接口
- 官方 OpenAI API
- OpenRouter

常见字段：

- Base URL：供应商接口地址
- 模型：图像生成模型名称
- API Key：供应商密钥

保存配置后，可以先点击测试连接。

### 3. 生成设计稿

1. 选择 `文生图` 或 `图生图`
2. 输入提示词
3. 选择比例，例如 `1:1`、`16:9`、`9:16` 或自定义尺寸
4. 选择图片数量
5. 点击 `生成设计稿`

生成完成后，右侧会展示结果。

### 4. 选择设计稿

如果生成多张设计稿，可以通过左右轮播按钮切换。选中满意的一张后，再进入切图模式。

### 5. 切图

点击 `切图` 后，在右侧设计稿上拖拽框选需要拆出的元素。

切图资产会显示在下方列表里，并保留：

- 名称
- 坐标
- 宽高
- 是否透明

### 6. 导入 Figma

点击 `放入 Figma`，插件会把当前设计稿和切图资产导入到 Figma 画布。

## 五、提示词建议

提示词可以直接描述你想要的界面：

```text
帮我生成一个年轻化的健身 App 首页，浅色风格，包含健康数据、训练计划、底部导航，9:16 手机屏
```

图生图时可以补充参考图用途：

```text
参考这张图片的布局和风格，帮我生成一个电商 App 首页，保留清晰的商品卡片和底部导航
```

如果你要切图，建议在提示词里强调：

```text
界面元素边界清晰，icon 和按钮不要互相遮挡，适合后续切图
```

## 六、常见问题

### 1. 保存配置失败

一般是本地 API 服务没有启动，先运行：

```bash
npm run api
```

### 2. 生成失败：Failed to fetch

可能原因：

- 本地 API 服务没有启动
- Base URL 填写错误
- Figma 插件访问不到本地代理
- 网络或供应商接口不可用

### 3. 生成超时

图像模型生成时间可能较长，尤其是高质量或多张图片。可以尝试：

- 降低生成数量
- 换供应商
- 检查本地代理是否设置了足够长的超时时间

### 4. 生成数量和返回数量不一致

不同供应商对批量生图的支持不完全一致。有些接口即使请求多张，也可能只返回一张合成图或一张结果。插件会尽量兼容返回格式，但最终数量仍取决于模型和供应商能力。

### 5. 图片比例不完全一致

不同图像模型对精确尺寸支持不一样。有些模型只支持固定比例，插件会尽量把用户选择的比例传给后端，但模型可能仍会按自己的规则输出。

### 6. 切图后背景修复不自然

当前版本主要支持 P0/P1 级别的背景填充修复，适合快速遮盖已切出的元素，但还不是专业级图像修复。后续可以继续优化为更自然的上下文修复。

## 七、当前版本边界

1.0 版本已经跑通核心流程，但仍有一些能力属于后续规划：

- 自动识别 UI 元素
- 自动生成切图建议
- 将 icon 转成可编辑 SVG
- 将整张设计稿还原为 Figma 可编辑图层
- 更高级的背景修复
- 更完整的资产命名和导出管理

## 八、推荐工作流

1. 输入提示词生成 2 到 4 张设计稿
2. 选择最满意的一张
3. 进入切图模式
4. 框选需要拆出的 icon、按钮和插画
5. 检查透明底效果
6. 点击 `放入 Figma`
7. 在 Figma 中继续整理和二次设计

## 九、项目定位

这个插件不是传统意义上的 UI 生成器，也不是单纯的图片下载器。它更像是一个 AI 视觉稿到 Figma 资产的工作流工具。

它当前最重要的价值是：

- 让 AI 生成结果进入 Figma
- 让生成图里的元素可以被拆出来
- 让切出的素材带着坐标回到设计画布
- 让设计师可以在 Figma 中继续接手，而不是停留在一张不可编辑图片上
