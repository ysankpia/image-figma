# Phase 1 执行方案：切图 + 透明 PNG + Figma 回填

## 1. 阶段目标

Phase 1 的目标不是直接做“完美自动还原设计稿”，而是先完成一个稳定的 Figma 内闭环：

> 生成 UI 图 -> 选择一张设计稿 -> 进入切图模式 -> 选择切图区域 -> 导出透明 PNG / 普通 PNG -> 保留坐标 -> 回填到 Figma 画布

这一阶段的重点是：

- 让用户能明确选择哪些区域需要切图
- 支持多张生成结果时，先选择一张设计稿再切图
- 让切出的素材能按原位置回到 Figma
- 让整个操作流程好看、顺手、可信

---

## 2. 功能范围

### 2.1 必做能力

- 在结果图上展示切图选择层
- 支持多张生成结果的设计稿选择
- 支持用户手动框选切图区域
- 支持常见预设切图类型
- 支持生成 `asset_manifest`
- 支持导出切图 PNG
- 支持将切图素材按原坐标放入 Figma
- 支持保留原始整图作为参考底图

### 2.2 暂不做能力

- 不做全自动完美识别所有 UI 元素
- 不做复杂遮挡关系恢复
- 不做整图完全可编辑化
- 不接 `DESIGN.md`
- 不考虑 Sketch

---

## 3. 推荐交互流程

### 3.1 生成结果

用户先通过文生图或图生图生成一张或多张 UI 图。

结果区显示：

- 主图预览
- 图片数量
- 当前比例
- 放入 Figma
- 下载
- 进入切图模式

UI 要求：

- 主图居中展示
- 不裁切，不变形
- 保留足够白边
- 操作按钮清晰但不抢主图

---

### 3.2 选择设计稿

如果一次生成了多张图片，用户需要先选择一张作为当前设计稿。

界面要求：

- 主图区域展示当前选中的设计稿
- 左右轮播按钮用于切换结果
- 顶部显示 `当前第 X / N 张`
- 只有当前选中的图片可以进入切图模式
- 进入切图模式后，切图数据绑定到当前图片

交互规则：

- 用户切换图片时，切图区域和资产列表要随之切换
- 每张生成图可以拥有独立的切图数据
- 如果某张图已经做过切图，再切回时要恢复它的选区
- 下载和放入 Figma 默认作用于当前选中的设计稿

数据要求：

```json
{
  "resultImages": [
    {
      "id": "image_1",
      "dataUrl": "data:image/png;base64,...",
      "sliceManifest": {
        "assets": []
      }
    },
    {
      "id": "image_2",
      "dataUrl": "data:image/png;base64,...",
      "sliceManifest": {
        "assets": []
      }
    }
  ],
  "activeImageId": "image_1"
}
```

这样可以避免一个常见错误：

> 用户生成了多张图，但插件默认拿第一张进入切图，导致用户切错设计稿。

---

### 3.3 进入切图模式

用户点击“切图”后，结果区进入可选择状态。

界面变化：

- 主图上出现轻量网格或半透明交互层
- 鼠标悬浮显示可选区域
- 用户可以拖拽框选区域
- 已选区域用描边和编号标记

推荐交互：

- 拖拽创建切图区域
- 点击区域选中
- 支持调整区域大小
- 支持删除区域
- 支持重命名区域

UI 要求：

- 框选层不能太重
- 选中态明确
- 编号、名称、类型要清楚
- 不要让画面像调试工具

---

### 3.4 资产面板

右侧或下方显示已选切图资产列表。

每个资产显示：

- 缩略图
- 名称
- 类型
- 尺寸
- 坐标
- 导出格式
- 是否透明底

可操作项：

- 重命名
- 删除
- 切换普通 PNG / 透明 PNG
- 勾选是否回填 Figma

UI 要求：

- 列表要像专业设计工具的图层/资产面板
- 不做大卡片堆叠
- 信息紧凑，可快速扫描

---

### 3.5 导出与回填

用户确认资产后，可以：

- 下载资产包
- 放入 Figma

放入 Figma 时：

- 创建一个新的 Frame
- 插入原始整图作为锁定参考图
- 将每个切图素材按 `x/y/width/height` 放回对应位置
- 给节点命名
- 给节点设置导出配置

Figma 节点建议：

- Frame 名：`AI UI Screen - {timestamp}`
- 原图层：`Reference / source_screen`
- 资产分组：`Assets`
- 单个资产名：`asset_{index}_{type}_{name}`

---

## 4. 数据结构

Phase 1 使用一个轻量版 manifest。需要注意：一次生成可能返回多张设计稿，因此切图数据必须绑定到具体图片，而不是绑定到整个生成任务。

```json
{
  "version": "1.0.0",
  "activeImageId": "image_1",
  "images": [
    {
      "id": "image_1",
      "screen": {
        "name": "generated_screen_1",
        "width": 1024,
        "height": 1536
      },
      "sourceImage": {
        "dataUrl": "data:image/png;base64,..."
      },
      "assets": [
        {
          "id": "asset_001",
          "name": "main_banner",
          "type": "banner",
          "bbox": {
            "x": 48,
            "y": 120,
            "width": 928,
            "height": 280
          },
          "export": {
            "format": "png",
            "transparent": false
          },
          "selected": true,
          "dataUrl": "data:image/png;base64,..."
        }
      ]
    }
  ]
}
```

这个结构后续可以升级为完整 `design manifest`。

## 5. 技术实现拆分

### Task 1：设计稿选择状态

文件：

- `ui.html`

内容：

- 维护 `activeImageId`
- 多张结果时支持左右切换
- 切换图片时恢复该图片自己的切图数据
- 下载、切图、放入 Figma 都默认作用于当前图片

验收：

- 生成多张结果时，用户能明确看到当前选中的是哪一张
- 进入切图模式前不会误用第一张图片
- 每张图片的选区互不覆盖

---

### Task 2：切图模式 UI

文件：

- `ui.html`

内容：

- 增加切图模式状态
- 在主图上叠加选择层
- 支持拖拽创建矩形区域
- 支持选中、删除、重命名

验收：

- 用户可以在图上框选多个区域
- 区域不会影响主图预览
- 区域坐标能准确记录

---

### Task 3：资产列表面板

文件：

- `ui.html`

内容：

- 展示已选区域列表
- 显示缩略图、名称、类型、尺寸
- 支持切换是否透明底
- 支持删除资产

验收：

- 选区和资产列表同步
- 删除列表项时主图选区同步消失

---

### Task 4：裁剪导出

文件：

- `ui.html`

内容：

- 用 canvas 按 bbox 裁剪源图
- 输出 PNG data URL
- 普通 PNG 先跑通
- 透明 PNG 可以先作为选项保留，后续接抠图能力

验收：

- 每个资产都能生成独立 PNG
- 下载后尺寸与 bbox 一致

---

### Task 5：Figma 回填

文件：

- `ui.html`
- `code.js`

内容：

- 将 manifest 发给 Figma 主线程
- Figma 创建 Frame
- 插入 reference image
- 插入每个 asset image
- 按 bbox 放置

验收：

- Figma 中能看到原图和资产图层
- 资产位置与原图一致
- 节点命名清楚

---

### Task 6：透明底能力接入

文件：

- `server.js`
- `ui.html`

内容：

- 针对单个 asset 调用透明图生成 / 抠图接口
- 返回透明 PNG
- 替换普通裁剪结果

验收：

- 透明 PNG 背景确实透明
- 坐标仍然保持不变

---

## 6. UI 设计要求

这一阶段的界面必须优先像一个专业 Figma 插件，而不是 demo。

### 6.1 总体风格

- 黑白灰为主
- 控制区清晰
- 预览区干净
- 不使用夸张装饰
- 不使用大面积彩色渐变

### 6.2 切图选择层

- 选区描边建议用黑色或深灰
- 选中态可以用蓝色强调
- 遮罩透明度要低
- 控制点要小而准

### 6.3 资产列表

- 使用紧凑列表
- 缩略图小尺寸
- 坐标和尺寸用等宽数字
- 操作按钮使用图标

### 6.4 状态反馈

- 生成中
- 切图中
- 导出中
- 放入 Figma 成功
- 放入 Figma 失败

这些状态必须明确显示，不能只在控制台里报错。

---

## 7. 关键风险

### 7.1 坐标换算错误

风险：

- 预览图缩放后，用户框选坐标可能与原图像素坐标不一致

应对：

- 所有选区都要从展示坐标换算回原图坐标
- manifest 只保存原图坐标

---

### 7.2 透明底质量不稳定

风险：

- 单纯裁切不能得到透明背景

应对：

- Phase 1 先支持普通 PNG
- 透明底作为可选增强
- 后续接专门抠图或图像分割能力

---

### 7.3 Figma 回填比例不一致

风险：

- 如果原图尺寸、预览尺寸、Figma Frame 尺寸不一致，资产位置会偏

应对：

- Figma Frame 必须使用 manifest 中的 screen 尺寸
- asset bbox 必须基于原图像素

---

## 8. 验收标准

Phase 1 完成时，至少需要满足：

1. 用户可以对生成图手动框选 3 个以上切图区域
2. 每个区域可以生成独立 PNG
3. 每个资产都有坐标、尺寸、名称
4. 点击“放入 Figma”后，原图和资产可以进入 Figma
5. 资产在 Figma 中的位置与原图视觉位置一致
6. UI 看起来像正式插件，而不是临时调试页

---

## 9. 推荐开发顺序

1. 先做切图模式 UI
2. 再做坐标换算
3. 再做资产列表
4. 再做普通 PNG 裁剪
5. 再做 Figma 回填
6. 最后接透明底增强

---

## 10. 一句话版本

Phase 1 的目标是：

> 先做一个好用、好看、坐标准确的 Figma 切图工具，再逐步增加自动识别和透明底能力。
